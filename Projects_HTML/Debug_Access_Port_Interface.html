Xilinx Debug Acces Port Interface

Main Concepts

  • Radiation effects including varying LET from different ions, energies, and fluence
  • ARM Debug Access Port Interface
  • Used for accessing Arm CoreSight Debug tools
  • Acces to global address space, coresight registers, and virtual UARTS
  • Much faster then SmartLynx tools as we can use our custom JTAG interface (JCM) to communicate at very fast speeds
  • Had to work at very low level handling bit control for both JTAG TAP controller and DAP interface controller shown in the diagram above

Materials Used

  • Microzed 7010 (JCM / JTAG)
  • Built custom CPP library to interface with controller
  • Utilizes JCM Drivers and firmware

Summary

This was a very challenging low-level software project that utilized the firmware of the JCM (JTAG interface). The project was tough due to poor documentation, either there was too little information, or it was so extensive that finding anything useful was nearly impossible. Additionally, it was difficult to track signals once they left the JCM over the JTAG cable, especially in the early stages.

Despite these challenges, I eventually got it to work, and it has proven to be very useful. Essentially, this project provides very low-level access to the board at runtime, including access to the CoreSight registers and global address space. With this access, I can query processor registers at runtime to check if the program counter (PC) has deviated from its expected path. I can also set up "virtual" UARTs for each processor to pull logs at runtime during a test. Moreover, with access to the global address space and the Network on Chip (NoC), I virtually have access to every part of the board.

I learned a lot while working on this project, including the importance of taking a slower, more methodical approach at times. Working with the bits and various controllers (TAP and DAP) was tedious and required exact precision. I also gained experience in building custom libraries in C++ and was able to abstract all the heavy lifting from the user. It's as simple as writing a JSON file with the registers you want to read or write at runtime, and with a click of a button, you have access to nearly the entire board. My work with this new feature has mainly involved the Xilinx Versal ACAP chip.
